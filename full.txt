I have already built a TypeScript agent that receives commands from a WebSocket server and executes them using `child_process.spawn()`. Simple commands like `pwd`, `ls`, and `echo` work correctly.

However, long-running commands such as:

* `python manage.py runserver`
* `cloudflared tunnel --url http://127.0.0.1:8000`
* `npm run dev`
* `node server.js`

either do not start correctly, exit immediately, or block the agent.

I want you to refactor my TypeScript agent to properly support long-running background processes.

### Requirements

1. Use `child_process.spawn()` instead of `exec()`.
2. Support both one-time commands and persistent processes.
3. Each spawned process should have a unique process ID stored in a `Map`.
4. Stream `stdout` and `stderr` back to the WebSocket in real time.
5. Send process status updates (`started`, `running`, `exited`, `error`) to the server.
6. Allow specifying a custom working directory (`cwd`) for commands like `python manage.py runserver`.
7. Preserve environment variables (`env: process.env`).
8. Do not block the WebSocket while a process is running.
9. Support running multiple processes simultaneously (for example, Django and Cloudflare Tunnel at the same time).
10. Add commands to:

    * start a process
    * stop a process
    * list running processes
    * restart a process
11. Handle Windows, Linux, and macOS correctly.
12. Ensure `python manage.py runserver` keeps running until explicitly stopped.
13. Ensure `cloudflared tunnel --url http://127.0.0.1:8000` also runs continuously and streams logs.
14. If the command requires an interactive shell, use the correct `shell` option.
15. Properly clean up child processes when the agent disconnects or exits.
16. Include complete TypeScript code with best practices and error handling.

Also explain why my current implementation fails for long-running commands and how your implementation fixes those issues.
